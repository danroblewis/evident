# runtime/src/portable/inject.rs — Z3-replaceability
**What it does:** The Rust shim for the FSM-membership injection pass, which auto-injects `state_next`, `last_results`, `effects`, `_var` time-shift slots, and `is_first_tick` into `fsm` schemas when those names are referenced but not declared. The identifier-walk over body expressions delegates to `inject_collect` in `stdlib/passes/inject.ev`; the actual BodyItem construction delegates to `fsm_params_build` and `prev_tick_build` FSMs in the same pass. Rust retains: `_`-prefix stripping and first-segment splitting (no substring ops in Evident at session PORT-inject), string-set membership tests (to avoid #18-class in-solve string-eq blowup), the `splice_at` insertion by `param_count`, and a self-hosted-pass exclusion guard.
**Criticality:** critical (load-time — runs on every `fsm` schema before translation; a bootstrapping guard prevents re-entrant engine loads)
**Verdict:** replaceable-as-group(portable/inject.rs, stdlib/passes/inject.ev)
**Confidence:** high
**How (if replaceable):** The walk and build steps are already self-hosted in `inject.ev`. Remaining Rust blockers identified at cut-over (session PORT-inject): (1) `_`-prefix strip and first-segment split require substring ops that Evident lacked at the time (now available via session GAPC's string ops); (2) `param_count` (the splice insertion index) was dropped by the marshaler (since fixed by SEED-marshal); (3) in-FSM boolean/string enum reads were wrong under #18. With GAPC string ops and SEED-marshal now landed, the substring and marshaler blockers are resolved — a follow-up session could attempt full cutover. The string-set membership guard would still need to stay in Rust or use the safe mirror-collect pattern.
**Change made:** none
