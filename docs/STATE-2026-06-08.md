# Evident — state & progress snapshot (2026-06-08)

A prose snapshot of where the project is and what's been accomplished.
Canonical state: `main` @ `a7110fc`.

---

## 1. The goal

Evident is a **constraint programming language**: programs are collections
of constraints over named variables, a Z3 SMT solver finds satisfying
assignments, and the central abstraction is the `schema`/`type`/`claim`/
`fsm`. Programs run as **multi-tick FSMs**: each tick the kernel solves the
constraint model, reads the resulting state, dispatches effects, and carries
state to the next tick.

The overarching goal is a **self-hosted, minimal, well-structured Evident
compiler** — the language compiling itself, on a tiny trusted runtime, with
the code organized well enough that the compiler is a showcase of the
language rather than an apology for it.

The producing path is:

```
source.ev ──▶ kernel + compiler ──▶ output.smt2 ──▶ kernel ──▶ exit / stdout
```

## 2. Architecture — the three layers

1. **The kernel** (`kernel/`, ~4.9k LOC Rust; ~2.6k is the runtime
   contract, ~2.3k is the functionizer accelerator). Parses the manifest,
   feeds the body to Z3 via `Z3_parse_smtlib2_string`, solves per tick,
   reads state, carries primitive top-level fields (`_<name>`), and
   dispatches effects (libffi/Z3/`__mem`). **Frozen in intent** — the
   architecture is complete; new language capability goes in the compiler
   or stdlib, not here. Diagnostics: `EVIDENT_PHASE_TRACE`,
   `EVIDENT_FUNCTIONIZE_STATS`, `EVIDENT_UNSAT_CORE`, `EVIDENT_EFFECT_TRACE`.

2. **The oracle** (`/usr/local/bin/evident-oracle`) — the build-time
   compiler. A frozen binary built from the deleted bootstrap Rust compiler,
   restored from pinned history *outside* the working tree, rebuilt, kept
   only as a binary. It compiles `compiler2/*.ev` → `.smt2`. The source
   never re-enters the repo. Pinned at commit `c95710c` (see §4 — the
   record-carry fix), rebuildable via `scripts/build-oracle.sh`. **Sunset
   target:** delete the day compiler2 compiles itself.

3. **compiler2** (`compiler2/*.ev`, ~7.8k lines / 29 files) — the
   self-hosted Evident compiler, the **Z3-AST-building** design: instead of
   concatenating SMT-LIB strings, it drives libz3 via `LibCall` to build a
   Z3 model and exports it. This is the active codebase and the corpus for
   everything below.

## 3. This session's arc (what was accomplished)

**Crash hunt & diagnostics.** Root-caused a `sample.ev` compile segfault to
a `Z3_mk_ite` with null then/else operands (a driver translation-logic bug),
using a new `EVIDENT_EFFECT_TRACE` diagnostic that logs every dispatched C
call + the per-tick state delta that caused it. Added a kernel null-AST
guard so such bugs surface as clean errors, not segfaults.

**Carry-preserving fsm composition.** Built (in the pre-oracle source
transform `expand-fsm-autocarry.sh`) the ability to compose `fsm`s so a
sub-fsm's `_x` carry threads into the parent — overturning the prior belief
that subsystems "can't own carry." Proven: a composed Counter prints
`0,1,2` across ticks; multi-counter, nested, and lift forms all work.

**driver_main decomposition.** Decomposed the 5930-line `driver_main`
monolith into **21 carry-owning fsm modules** (`driver_lex`, `driver_expr`,
`driver_record`, `driver_enum`, `driver_compose`, …), each with a
CONSUMES/PRODUCES/MAINTAINS contract header and an isolation test — **28
unit fixtures**, every step equivalence-gated (byte-identical emit modulo
the oracle's call-counter), conformance held at **137/138**. `driver_main`
is now **1538 lines** (−74%). It stopped at the honest quality floor: the
work-item lowering engine is genuinely coupled and forcing it into modules
would create the wide-interface/weak-test anti-pattern the standards reject.

**Discovery: analyzer + taxonomy.** Two complementary explorations,
merged to `main`:
- A **mechanical analyzer** (`scripts/analyze-abstractions.sh` +
  `docs/analysis/hidden-abstractions-report.md`) — frequency-ranked hidden
  types, reusable-claim shapes (the **carry-Latch: 348 sites, no
  abstraction**; build-capture 119; raw LibCall 171), and subclaim seams.
- A **design-pattern taxonomy** (`docs/analysis/constraint-programming-
  patterns.md`, ~30 GoF-style patterns) with web prior-art research. Key
  finding: constraint-modeling pattern catalogs exist *only for encoding
  search problems for a solver*; **FSM-over-SMT — driving deterministic
  stateful computation by re-solving each tick — is uncharted territory.**

**Oracle record-carry fix.** Records compiled as Z3 tuples but **didn't
carry across ticks**, blocking the type refactor. Fixed (with explicit,
one-time permission to edit the recovered bootstrap Rust; kernel untouched):
two bugs in the oracle's `emit.rs::discover_state_fields` — pinned constants
wrongly listed as carries, and dotted record-field carries skipped. The fix
flattens a carried record into its primitive field-consts (the "types
convert down to primitive numbers" model). Verified: record-from-variable,
single- and multi-field record carry all work; conformance 137/138, stage1
byte-identical. The fixed bootstrap is preserved in history (`c95710c`),
absent from the tree.

## 4. Capabilities & constraints established (the hard-won facts)

- **Records now carry** as state: declare `x ∈ T` + `_x ∈ T`; fields carry
  as primitives. (Enables the type refactor.)
- **Compound-state carry hierarchy:** primitives ✓, cons-list enums ✓
  (the fast choice the codebase uses), `Seq(T)` ✓ but **~250× slower on
  Z3** (avoided), records ✓ (just fixed). → Use **record types for
  elements, cons-list enums for collections** — never `Seq` for hot
  collections.
- **`fsm` composition carries** (slot-bind injects `_x ↦ _y`; `..`-lift and
  nested forms work).
- **`..`-lift is byte-identical** to inline; slot-bind composition churns
  the emit. Both preserve behavior.
- The **carry model shapes the code**: flattened arrays-of-scalars (`rt_n0`/
  `rt_n1`/…) are the forced encoding of carried compound state, not
  sloppiness.

## 8. Infrastructure & process

- **Goalpost system** (`.goalpost/`): independent measure scripts compute
  progress live (conformance, kernel-fixtures, sample, **code-quality**:
  max/p95/oversized claim size). The conformance gate (137/138) is the
  behavior-preservation guard for every refactor.
- **Orchestration pattern** (learned the hard way): refactor agents run
  pinned to dedicated `/tmp/*-wt` worktrees (never the shared checkout —
  agents that `cd` into it have clobbered `main` twice; recovered both
  times). An **integrity monitor** cron checks `main` every 11 min and
  auto-reverts + stops any straying agent. Work is staged-committed per
  green step (agents are killed ~30 min in; continuations resume from the
  branch), and nothing auto-merges — branches are reviewed before landing.
- **Backups**: `main` is pushed to GitHub (operator's machines); the
  container is disposable, so worst case is wasted compute, not lost work.
