# Diagram review — overall behavior vs simulation run

_Generated from a 22-agent parallel review (one per diagram type). Each agent read its `viz/render_<view>.py`, researched the diagram's standard form, rendered it across the IDE samples, visually inspected the PNGs, and judged whether it shows ALL-initial-conditions behavior or a single simulation run._

## The systemic finding

Most dynamics renderers seed/root from `initial_state()` (the single `is_first_tick=true` state) via `reachable()`, then draw what's forward-reachable from that ONE seed. For a deterministic FSM that's a single trajectory — exactly the 'renders a simulation, not the model's behavior for all initial conditions' problem. The fix is cross-cutting: seed/sweep ALL valid starting states (the full state product for discrete, a state-space grid for continuous), not the forward orbit of one init. The `function_*` views score well precisely because they render the COMPILED TRANSFER FUNCTION, which is inherently all-conditions.

## Verdict (shows_all_conditions)

- **NO — renders a single simulation run (7):** basin_map, fixedpoint_map, orbit_scatter, reachability_tree, state_graph, time_series, timing_diagram
- **PARTIAL (10):** chord_diagram, cobweb, function_behavior, morse_graph, occupancy_heatmap, parallel_coords, phase_portrait, scatter_matrix, solution_space, transition_matrix
- **YES — shows overall behavior (5):** function_complexity, function_graph, function_guards, function_residual, nullcline_field
